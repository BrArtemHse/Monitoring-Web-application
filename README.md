Проект реализует простую систему мониторинга веб-приложения.
Система состоит из двух компонентов:

1. Веб-приложение (Go)

Лёгкий HTTP-сервер, который, отвечает "Hello World!" на корневом маршруте и имеет эндпоинт /health для проверки доступности.

2. Скрипт мониторинга (Go)

Мониторинг работает как отдельный процесс и выполняет:
периодическую проверку доступности веб-приложения,
логирование результатов проверки,
перезапуск приложения при недоступности или при падении процесса,
автоматическую загрузку параметров из конфигурационного файла.
Все параметры вынесены в config.json:
команда запуска приложения,
URL для health-check,
интервал проверок,
путь к лог-файлу,
порог ошибок до перезапуска.

3. Docker / Docker Compose

Для автоматизации используется Docker:
сборка обоих Go-бинарников в одном образе,
запуск мониторинга как основного процесса,
запуск веб-приложения как дочернего процесса,
автоматический рестарт контейнера при сбоях (restart: always),
возможность обновления одной командой docker compose up --build.

Как запустить
1. Собрать образ
docker compose build

2. Запустить контейнеры
docker compose up -d


После запуска:

веб-приложение доступно на http://localhost:8080/,
мониторинг работает внутри контейнера как основной процесс,
лог-файл доступен внутри /var/log/monitor.log и на на хост-машине в директори проекта в папке /logs.

Проверка логов
Внутри контейнера:
docker exec -it monitoring-app sh
cat /var/log/monitor.log

Как обновить систему

При изменении кода или конфигурации:
docker compose up -d --build


Docker пересоберёт бинарники и перезапустит систему.

Завершение
docker compose down
